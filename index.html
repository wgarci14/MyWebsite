Brief Summary:
ARIA consists of a remote control vehicle with various sensors and microcontroller board attached, Arduino Uno R3, that transmits data back to the Desktop Application Module (DAM) to manage the machine learning aspects of the project. Arduino Uno R3 simulation and programming can be easily translated to instructions for various types of hardware, such as satellites. The system’s software for ARIA is the DAM that runs on the server to provide several modes and features to the Elegoo vehicle.

The software offers four types of modes: Training Mode, Testing Mode, Manual Mode, and Autopilot Mode. The Training Mode allows the vehicle to read real-time data as it records data on anomalies. The Testing Mode allows the vehicle to run test trials based on the stored data to learn how to behave in certain situations. The Manual Mode allows the vehicle to be fully controlled by the user through a user-interface on the DAM. The Autopilot Mode allows the vehicle to run autonomously based on the testing data.

The Vehicle Control Module (VCM) is the software on the Arduino Uno R3. It contains various libraries and methods that are provided by Elegoo to convert raw data from the sensors into actual data. The VCM sends actual data to the DAM in bytes and the DAM converts the bytes into a Java String array and the strings will be parsed to primitive data types (double and boolean). The VCM reads user input as a character from the DAM to move the vehicle forward, backward, left, and right depending on the character that is given.

In a more layman’s description of machine learning, the software will allow the car to improvise, adapt, overcome situations that it was not explicitly programmed to do ( e.g. It will come to its own conclusion that continuously driving into a wall is bad).

By simple deduction, this also means that the software will not be structured with predetermined responses to some actions ( e.g. it will not be programmed to simply avoid walls ). This type of setup allows for the software to learn from mistakes, so as to readily understand when and where mistakes may be made in the future.
